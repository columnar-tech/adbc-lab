Exercise 2: Work locally with DuckDB
=====================================

You pulled some interesting data from Postgres. Now you want to save it locally
so you can keep exploring without needing the database connection — maybe on
your laptop, maybe offline.

Part 1: Install the DuckDB driver
-----------------------------------

    dbc install duckdb


Part 2: Move data from Postgres to DuckDB
------------------------------------------

Open exercise_2.py. The script pulls all trip data from Postgres and loads it
into a local DuckDB file using adbc_ingest().

It then queries Postgres — but the DuckDB query section is incomplete.
Fill in the blanks to run the same query against DuckDB.

Hint: look at the Postgres query block just above — the DuckDB version is
almost identical.


Part 3: Run it
--------------

    uv run exercise_2.py

- Are the results the same across both backends?
- What did adbc_ingest() do? Did you need to write any SQL to create the table?
- How much of the code had to change to switch from Postgres to DuckDB?
