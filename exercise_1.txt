Exercise 1: Install ADBC and run your first query
==================================================

You're an analyst. There's a Postgres database with NYC Citi Bike trip data —
about a million rows of trips with timestamps, stations, durations, and user info.
You want to pull some data down to explore.

Part 1: Install dbc
--------------------

dbc is a CLI tool for installing ADBC drivers. Install it with:

    uv tool install dbc

Try running these to see what's available:

    dbc search
    dbc info postgresql


Part 2: Install the PostgreSQL driver
--------------------------------------

    dbc install postgresql


Part 3: Run your first ADBC query
-----------------------------------

Open exercise_1.py. The query is written but the ADBC connection and fetch
calls have blanks. Fill them in, then run it:

    uv run exercise_1.py


Go further
----------

If you've got time, try these:

- Change fetch_arrow_table() to fetchall() — what do you get instead?
- Change fetch_arrow_table() to fetch_df() — what do you get?


Extension: See ADBC in action
-----------------------------

You saw the Streamlit demo earlier — now run it yourself. This app queries the
same Citi Bike data, but lets you toggle between psycopg2 (row-based) and ADBC
(columnar) and see the speed difference live.

Run it with:

    uv run --with "psycopg2-binary, adbc-driver-manager, streamlit, pydeck, pandas" streamlit run demo/app.py

Things to try:

- Toggle between psycopg2 and ADBC and compare the query times
- Try different numbers of trips — how does the speed gap change as you scale up?
